# 전투력 효율 시뮬레이터 설계

Date: 2026-05-20

## Goal

`전투력 효율 시뮬레이터`를 상위 기능으로 분리하고, 첫 활성 모듈로 `악세사리 교체 추천`을 만든다.

이 기능은 경매장 악세사리 후보를 현재 착용 악세와 비교해 `전투력 +1%당 필요한 골드`를 보여준다. 첫 버전은 정확도가 중요한 구매 판단 도구로 설계하며, 계산 근거가 약한 값은 추천 지표처럼 노출하지 않는다.

## Current Context

현재 프로젝트는 Next.js API Route가 공식 Lostark Open API를 호출하고, JS 모듈에서 장비 정규화와 계산을 수행한다.

- `app/api/characters/[name]/route.js`는 캐릭터 장비, 각인, 보석, 카드, 아크 패시브를 불러오고 계산 DTO를 반환한다.
- `lib/lostark/equipment.js`는 Tooltip 기반 장비 정규화를 담당한다.
- `lib/spec/combatPowerModel.js`는 내부 전투력 추정 모델을 제공한다.
- `lib/spec/accessoryContributions.js`는 목걸이/귀걸이/반지 연마 효과의 피해 기여도를 계산한다.
- `lib/lostark/marketSnapshot.js`와 `lib/lostark/marketApi.js`는 거래소/경매장 스냅샷을 가져온다.

Spring Boot BFF는 별도 전환 단계가 진행 중이지만, 이번 기능은 기존 JS 계산 모듈을 재사용하기 위해 Next.js API Route에 먼저 구현한다.

## Scope

이번 설계에 포함한다.

- 상위 메뉴 `캐릭터 분석 | 전투력 효율 시뮬레이터`.
- 시뮬레이터 페이지의 첫 활성 모듈 `악세사리 교체 추천`.
- T4 고대 목걸이/귀걸이/반지 후보 탐색.
- 깨달음 포인트 MAX 품질 이상 후보만 비교.
- 후보 악세를 현재 장비에 가상 장착한 전투력 상승 계산.
- `전투력 +1%당 골드` 기준 1등 추천과 상위 3개 접힌 비교.
- 빠른 추천 API와 현재 악세 회수가 추정 API 분리.
- 회수가 신뢰도가 충분할 때만 순비용 기준 효율 표시.
- fixture 기반 테스트로 탐색, 정규화, 교체 계산, 회수가 추정을 검증.

이번 설계에서 제외한다.

- 팔찌 효율 계산. 팔찌는 별도 예정 모듈로 분리한다.
- 보석, 아바타, 강화 후보를 같은 랭킹에 섞는 기능.
- Spring Boot로 계산 로직 이전.
- 스킬별 딜 지분 입력.
- 스킬 전용/직업 조건부 치명타 피해를 전체 피해 효율에 자동 반영.
- 현재 악세를 실제로 판매하는 기능이나 거래 자동화.

## Product Structure

제품 페이지 이름은 `전투력 효율 시뮬레이터`다.

첫 릴리스의 활성 모듈은 `악세사리 교체 추천`이다. 페이지 구조는 이후 보석, 아바타, 강화, 팔찌 같은 성장 수단을 모듈로 추가할 수 있게 잡는다. 다만 첫 구현에서는 악세사리 추천만 활성화하고 다른 성장 수단은 같은 추천 랭킹에 섞지 않는다.

팔찌는 일반 악세사리와 옵션 체계가 다르므로 `팔찌 효율 시뮬레이터` 예정 모듈로 분리한다. 목걸이/귀걸이/반지 계산 로직에 팔찌를 억지로 포함하지 않는다.

## User Flow

사용자는 두 방식으로 시뮬레이터를 사용할 수 있다.

- 캐릭터 분석 페이지에서 이미 조회한 캐릭터가 있으면 그 캐릭터 컨텍스트를 재사용한다.
- 시뮬레이터 페이지에 직접 들어온 경우, 페이지 안에서 캐릭터명을 입력해 바로 분석한다.

시뮬레이터는 먼저 빠른 추천 결과를 보여준다.

1. 캐릭터 장비와 현재 공식 전투력을 불러온다.
2. T4 고대 목걸이/귀걸이/반지 후보를 경매장에서 탐색한다.
3. 후보를 현재 장비에 가상 장착하고 내부 전투력 계산기로 상승분을 계산한다.
4. `전투력 +1%당 골드`가 가장 낮은 후보 1개를 메인 카드로 보여준다.
5. 상위 3개 후보는 접힌 비교 목록으로 제공한다.
6. 회수가 추정 API가 별도로 실행되며, 완료 후 순비용 참고값을 갱신한다.

추천 후보가 없으면 `현재 조건에서 추천할 만한 악세가 없습니다`만 표시한다.

## Recommendation Metric

기본 추천 지표는 `전투력 +1%당 골드`다.

```text
전투력 상승률(%) = 내부 계산 상승분 / 현재 공식 전투력 * 100
전투력 +1%당 골드 = 후보 즉시구매가 / 전투력 상승률(%)
```

낮을수록 좋은 후보다. 추천 정렬은 피해 효율이 아니라 이 지표로 결정한다.

예상 전투력은 다음 방식으로 표시한다.

```text
예상 전투력 = 현재 공식 전투력 + 내부 계산기로 산출한 후보 교체 상승분
```

내부 계산기의 후보 장착 후 원값은 MVP 사용자 화면에 노출하지 않는다. 현재 공식 전투력과 이어지는 예상값만 보여줘 신뢰 혼선을 줄인다.

## Candidate Replacement Rules

후보 비교는 같은 부위 안에서 수행한다.

- 목걸이 후보는 현재 목걸이와 비교한다.
- 귀걸이 후보는 현재 착용 중인 두 귀걸이에 각각 가상 교체해 보고 더 좋은 교체 대상을 선택한다.
- 반지 후보도 현재 착용 중인 두 반지에 각각 가상 교체해 보고 더 좋은 교체 대상을 선택한다.

추천 카드에는 교체 대상 현재 악세와 후보 악세를 나란히 보여준다.

표시 필드:

- 현재 악세 이름, 품질, 주요 옵션, 깨달음 포인트.
- 후보 악세 이름, 품질, 주요 옵션, 깨달음 포인트.
- 후보 즉시구매가.
- 현재 공식 전투력.
- 예상 전투력.
- 전투력 상승분과 상승률.
- 즉시구매가 기준 `전투력 +1%당 골드`.
- 회수가 추정이 충분히 신뢰될 때의 순비용 기준 `전투력 +1%당 골드`.

## Auction Search Policy

검색 대상은 T4 고대 목걸이/귀걸이/반지로 고정한다.

탐색은 가격순 페이지 방식이다. 부위별로 즉시구매가가 낮은 페이지부터 가져와 후보를 계산한다. 단순히 최저가 30개만 보는 방식은 사용하지 않는다. 첫 구현은 지정된 탐색 한도까지 가격 구간을 넓혀가며 확인하고, API가 더 이상 매물을 반환하지 않거나 최대 페이지/후보 수에 도달하면 중단한다.

정확도와 API 비용을 함께 관리하기 위해 구현에는 명시적 한도를 둔다.

- 부위별 최소 탐색: 3페이지.
- 부위별 최대 탐색: 10페이지 또는 raw 후보 100개 중 먼저 도달하는 값.
- 빠른 추천 API hard timeout: 12초.
- 회수가 추정 API hard timeout: 15초.
- 강제 새로고침 시 캐시 우회.

MVP의 조기 중단은 안전한 조건에만 적용한다. API가 빈 페이지를 반환하거나 TotalCount 기준 마지막 페이지에 도달하면 중단한다. 점수 기반 조기 중단은 늦은 페이지의 고효율 매물을 놓칠 수 있으므로 첫 구현에는 넣지 않는다. 한도값은 상수로 분리해 운영 중 조정할 수 있어야 한다.

## Quality And Enlightenment Policy

후보는 깨달음 포인트 MAX가 나오는 품질 이상만 비교한다.

품질 필터는 두 단계다.

1. 부위별 깨달음 포인트 MAX 최소 품질 기준값으로 1차 필터링한다.
2. 후보 Tooltip 또는 Options에서 아크 패시브 포인트 효과를 파싱해 실제 MAX 여부를 검증한다.

검증 실패 후보는 추천 대상에서 제외한다. 기준값은 설정 테이블로 분리해 게임 패치 시 수정 가능하게 한다.

현재 착용 악세가 기준 미만이어도 후보는 기준 이상만 추천한다. 화면에는 `최대 깨달음 포인트 품질 이상 매물만 비교`한다는 기준을 명확히 표시한다.

## Damage Efficiency Role

피해 기대값은 참고 정보다. 추천 순위를 결정하지 않는다.

첫 버전에서 피해 효율은 전역 적용 옵션만 계산한다.

- 추가 피해.
- 적에게 주는 피해.
- 공격력.
- 무기 공격력.
- 치명타 적중률.
- 전역 치명타 피해.

스킬 전용 치명타 피해, 특정 스킬군 효과, 직업 조건부 효과는 전체 피해 기대값에 자동 반영하지 않는다. 이런 옵션은 `계산 제외/확인 필요`로 표시한다. 모르는 값을 억지로 계산해 추천 점수를 왜곡하지 않는다.

## API Design

API는 두 단계로 분리한다.

### Fast Recommendation API

역할:

- 캐릭터 컨텍스트 조회.
- 현재 장비 fingerprint 생성.
- 경매장 후보 탐색.
- 후보 정규화.
- 가상 교체 전투력 계산.
- 1등 추천과 상위 3개 반환.

응답에는 즉시구매가 기준 결과만 포함한다. 회수가 추정 상태는 `pending`으로 둘 수 있다.

### Recovery Estimate API

역할:

- 빠른 추천 결과의 교체 대상 현재 악세를 기준으로 유사 매물을 탐색한다.
- 같은 부위, 티어, 등급, 품질 구간, 주요 연마 옵션 종류, 옵션 수치 등급, 옵션 조합 유사성을 비교한다.
- 예상 회수가, 근거 매물 수, 신뢰도를 반환한다.

회수가 추정의 기본 정책:

- 같은 부위 T4 고대 매물만 비교한다.
- 품질은 현재 악세와 같은 깨달음 포인트 구간 안에서 비교한다.
- 주요 연마 옵션 버킷과 수치 등급이 모두 일치하는 매물을 exact match로 본다.
- exact match가 3개 이상이고, 매물 가격의 사분위 범위가 중앙값의 35% 이하일 때만 신뢰도 충분으로 판정한다.
- 예상 회수가는 exact match 즉시구매가의 중앙값을 사용한다.
- 조건을 만족하지 못하면 신뢰도 낮음으로 판정한다.

신뢰도가 충분하면 순비용 기준 효율을 표시한다.

```text
순비용 = 후보 즉시구매가 - 현재 악세 예상 회수가
순비용 기준 전투력 +1%당 골드 = 순비용 / 전투력 상승률(%)
```

신뢰도가 낮으면 순비용 효율은 숨긴다. 숫자가 있어도 구매 판단 지표처럼 보이지 않게 한다.

## Cache Policy

캐시는 원본 경매장 페이지와 캐릭터별 계산 결과를 분리한다.

- 경매장 원본 페이지 캐시: 1-3분 TTL.
- 캐릭터별 추천 계산 결과: 현재 장비 fingerprint 기반의 더 짧은 TTL.
- 강제 새로고침: 원본 페이지 캐시와 계산 결과 캐시를 모두 우회.
- 화면에는 경매장 가격 기준 시각을 표시한다.

장비 fingerprint에는 적어도 현재 악세의 부위, 이름, 품질, 주요 옵션, 깨달음 포인트, 전투력 계산에 필요한 정규화 결과가 포함되어야 한다.

## UI States

시뮬레이터 페이지의 주요 상태는 다음과 같다.

- 초기 상태: 캐릭터명 입력 또는 최근 조회 캐릭터 재사용.
- 캐릭터 로딩: 공식 API에서 장비와 전투력 조회 중.
- 빠른 추천 로딩: 경매장 후보 탐색과 전투력 계산 중.
- 추천 있음: 1등 추천 카드와 상위 3개 접힌 비교 목록.
- 회수가 계산 중: 현재 악세 예상 회수가 계산 중.
- 회수가 신뢰도 충분: 예상 회수가, 순비용, 순비용 기준 효율 표시.
- 회수가 신뢰도 낮음: 근거 부족 안내, 순비용 기준 효율 숨김.
- 추천 없음: `현재 조건에서 추천할 만한 악세가 없습니다`.
- 오류: API 키 누락, 캐릭터 없음, 경매장 응답 불안정, 계산 불가를 구분.

상위 3개 접힌 비교 목록의 기본 표시 필드는 후보 이름, 부위, 교체 대상, 즉시구매가, 전투력 상승률, 즉시구매가 기준 `전투력 +1%당 골드`다. 펼친 상세에서 품질, 깨달음 포인트, 주요 옵션, 피해 기대값 참고값을 보여준다.

## Testing Strategy

네트워크 없는 fixture 기반 테스트를 만든다.

필수 테스트 범위:

- 경매장 악세 후보 응답 정규화.
- 후보 Tooltip/Options에서 기본 효과, 연마 효과, 아크 패시브 포인트 파싱.
- 깨달음 MAX 품질 기준 1차 필터.
- 실제 아크 패시브 포인트 MAX 검증.
- 목걸이 단일 교체 계산.
- 귀걸이/반지 두 슬롯 중 최적 교체 대상 선택.
- 전투력 상승률 계산.
- `전투력 +1%당 골드` 정렬.
- 1등 추천과 상위 3개 후보 반환.
- 추천 후보 없음 상태.
- 가격순 페이지 탐색과 조기 중단.
- 경매장 원본 페이지 캐시와 계산 결과 캐시 분리.
- 회수가 추정 매칭: 품질 구간, 옵션 종류, 옵션 수치 등급, 조합 유사성.
- 회수가 신뢰도 충분/낮음 분기.
- 신뢰도 낮을 때 순비용 효율 숨김.

기존 회귀 테스트도 유지한다.

- `tests/accessoryContributions.test.js`
- `tests/combatPowerModel.test.js`
- `tests/marketSnapshot.test.js`

## Acceptance Criteria

작업은 다음 조건을 만족해야 완료로 본다.

- `전투력 효율 시뮬레이터`가 상위 메뉴로 분리되어 접근 가능하다.
- 시뮬레이터 페이지에서 캐릭터명 직접 입력과 이전 조회 캐릭터 재사용이 가능하다.
- 첫 활성 모듈은 목걸이/귀걸이/반지 악세사리 교체 추천이다.
- 팔찌는 별도 예정 모듈로 분리되어 일반 악세 계산에 포함되지 않는다.
- 후보 검색은 T4 고대 목걸이/귀걸이/반지만 대상으로 한다.
- 깨달음 MAX 품질 이상 후보만 추천 계산에 포함된다.
- 후보를 가상 장착한 전투력 상승분으로 예상 전투력을 표시한다.
- 기본 추천 지표는 즉시구매가 기준 `전투력 +1%당 골드`다.
- 메인 화면은 1등 추천 하나를 강조하고, 상위 3개는 접힌 비교 목록으로 제공한다.
- 회수가 추정은 별도 API로 수행된다.
- 회수가 신뢰도가 낮으면 순비용 기준 효율을 표시하지 않는다.
- 추천 후보가 없으면 `현재 조건에서 추천할 만한 악세가 없습니다`를 표시한다.
- fixture 기반 테스트가 경매장 탐색, 후보 정규화, 교체 계산, 회수가 추정을 검증한다.

## Deferred Decisions

다음 항목은 이번 설계의 완료 조건이 아니며, 별도 설계 후 진행한다.

- 팔찌 효율 시뮬레이터 구현.
- 보석, 아바타, 강화 모듈의 추가 순서.
- 점수 기반 조기 중단 도입 여부.
- 내부 전투력 계산기의 후보 장착 후 원값을 사용자 화면에 공개할지 여부.
